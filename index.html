⚙ Features — everything in the core
⚡
Direct P2P transport

Messages and files ride a WebRTC datachannel opened directly between two devices — no message server in between.

📡
Blind rendezvous

A relay board only helps two peers find each other. It can't read traffic and stores nothing once you're connected.

▦
Identity via QR

Add contacts by scanning a QR identity card, with anti-substitution binding validation on every scan. Requests & blocking built in.

🛡
Recipient-sealed messages

Every payload is sealed to your contact's verified key — readable by them, and by no one else.

📦
Chunked file transfer

Large files stream in verified chunks, each checked with SHA-256, so what lands is exactly what was sent.

💾
Encrypted backup & restore

A full device backup, encrypted end to end. Restore your identity and history on a new device with confidence.

Reliable delivery — with an offline send queue

Peer-to-peer doesn't mean fragile. Compose anytime; messages wait safely on your device and send the instant a peer reconnects.

🔐 Security — Properties

Designed to be trusted with the relay assumed hostile.

Hardware keys
🔑
Keys in secure hardware

When the device supports it, private keys are sealed to secure hardware — Secure Enclave, Android Keystore, or TPM via WebAuthn — and never leave it.

Untrusted board
🚫
The board is untrusted

Orpal assumes the relay is fully hostile. Even a compromised board learns nothing — no keys, no plaintext, no usable connection data.

Open source
📖
Apache-2.0, fully auditable

Every line is open. The full client and the Open Relay Protocol are released under Apache-2.0 — read it, fork it, run your own board.
